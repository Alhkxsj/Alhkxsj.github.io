🚀 GitHub Pages 免费部署静态网站

一、什么是 GitHub Pages？

GitHub Pages 是 GitHub 官方提供的免费静态网页托管服务，无需服务器，使用你的仓库代码即可一键上线个人网站。

二、创建用于部署的网站仓库

  1. 登录 GitHub，点击右上角 + → New repository。
  2. 仓库名称建议为 用户名.github.io（如 Alhkxsj.github.io），这样可直接用于个人主页。
  3. 选 Public，可选 Add a README。
  4. 创建后，克隆仓库到本地，或用 Termux/电脑编辑网页内容。

三、上传你的网页文件

  1. 确保仓库根目录有 index.html，可直接用之前制作的网页。
  2. 用 git 命令提交并推送：
    git add .
git commit -m "发布我的网站"
git push

四、开启 GitHub Pages 服务

  1. 进入你的仓库主页，点击 Settings → Pages（侧边栏，或在 "Code and automation" 部分）。
  2. 在 Build and deployment 区块，Source 选择 Deploy from a branch。
  3. 分支选 main，文件夹选 /(root)。
  4. 保存后，页面上方会出现网站访问链接，如 https://用户名.github.io/。

五、绑定自定义域名（可选）

  1. 在 Settings → Pages 页面，填写你的自定义域名，保存。
  2. 到你的域名管理后台，添加一条 CNAME 记录，指向 用户名.github.io。
  3. 等待生效后，GitHub Pages 会自动配置 HTTPS。

六、常见问题

  • Q: 网页打不开？
    A: 检查 index.html 是否在仓库根目录，稍等几分钟再试，或清空浏览器缓存。
  • Q: 域名无法绑定？
    A: 检查 DNS 是否正确设置了 CNAME，且仓库的 GitHub Pages 域名填写无误。
  • Q: 样式/图片加载不出来？
    A: 路径要用相对路径，注意区分大小写，确保相关文件已推送到仓库。

七、进阶玩法

  • 使用 Actions 自动部署 Hexo、Hugo、VuePress 等静态博客。
  • 多页网站可用子文件夹或多 HTML 文件。
  • 推荐结合 官方文档 深入学习。
