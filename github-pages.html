一、什么是 GitHub Pages？

GitHub Pages 是 GitHub 官方提供的免费静态网站托管服务，无需服务器，推送代码即可上线，支持自定义域名和 HTTPS。

二、准备工作

  1. 完成 环境配置（见左侧/上方导航），确保你已经可以用 git 正常推送仓库。
  2. 注册 GitHub 账号并新建一个公开仓库，推荐命名为 你的用户名.github.io，如 Alhkxsj.github.io。

三、上传并部署你的网页

  1. 将你的 HTML/CSS/图片等静态文件放入本地仓库文件夹。
  2. 使用 git add .、git commit -m "部署网站"、git push 推送到 GitHub。
  3. 首次推送后，进入仓库 Settings > Pages，将 Source 设为 main 分支（或 master），保存。
  4. 稍等 1-3 分钟，访问 https://你的用户名.github.io/，即可看到你的个人网站上线！

四、常见问题与排查

  • Q: 网页打不开或 404？
    A: 检查仓库名是否为 你的用户名.github.io，主分支是否有 index.html 文件，Pages 是否已启用。
  • Q: 推送后页面没变化？
    A: 通常缓存约 1-3 分钟，Ctrl+F5 强刷即可。如无效可到 Pages 设置页面点 Rebuild 或检查 commit 是否成功。
  • Q: 如何绑定自定义域名？
    A: 在 Pages 设置页面添加自定义域名，并到域名服务商设置 CNAME 解析指向 你的用户名.github.io。

五、进阶玩法

  • 配合 Jekyll/Hexo/Hugo 等静态博客框架实现自动化博客部署。
  • 用 Actions 实现自动构建和部署。
  • 用 GitHub Pages 托管简历、作品集、文档站等。
