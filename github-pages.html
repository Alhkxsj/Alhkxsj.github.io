🚀 GitHub Pages部署详解

一、什么是GitHub Pages？

GitHub Pages是GitHub提供的免费静态网站托管服务，非常适合个人主页、项目展示、博客等场景。

二、创建仓库

  1. 登录你的GitHub账号，新建一个仓库，命名必须为 用户名.github.io（用户名就是你的GitHub用户名）。
  2. 建议勾选“Initialize this repository with a README”。

三、上传网页文件

  1. 在Termux中克隆仓库： 
    git clone git@github.com:你的用户名/你的用户名.github.io.git
  2. 将你的 index.html、style.css 等网页文件复制到仓库目录下。
  3. 提交并推送到GitHub： 
    
git add .
git commit -m "初始化网站"
git push

四、访问你的网站

在浏览器输入：
https://你的用户名.github.io
等几分钟后刷新即可看到你的网站上线。

五、自定义域名（可选）

  1. 在仓库根目录新建一个名为 CNAME 的文件，内容写上你的自定义域名（如 www.example.com）。
  2. 在你的域名服务商处添加CNAME解析，指向 你的用户名.github.io。

六、常见问题

  • Q: 网站打不开？
    A: 检查仓库名是否为 用户名.github.io，index.html 是否在仓库根目录，并确认已推送到GitHub。
  • Q: 页面长时间未更新？
    A: 清除浏览器缓存，或在GitHub Pages设置中重新构建。
  • Q: 自定义域名配置无效？
    A: 检查CNAME文件内容是否正确，域名解析是否生效，等待DNS生效时间。
