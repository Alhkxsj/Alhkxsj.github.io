🚀 GitHub Pages 部署全解

一、什么是 GitHub Pages？

GitHub Pages 是 GitHub 官方提供的 免费静态网页托管服务，适合搭建个人主页、项目演示、作品集等。无需服务器、无需备案，全球可访问，一键部署，适合新手快速上手建站。

二、创建站点专用仓库

  1. 登录你的 GitHub 账号，点击右上角 New repository 创建新仓库。
  2. 仓库名称必须为 你的用户名.github.io。
    例如用户名 Alhkxsj，仓库名就为 Alhkxsj.github.io。
  3. 建议勾选 Initialize this repository with a README。
  4. 可选：添加 .gitignore 和 LICENSE。

三、上传网页文件

  1. 用 Termux 或 PC 克隆仓库： 
    git clone git@github.com:你的用户名/你的用户名.github.io.git
  2. 将你的 index.html、style.css、图片等网页文件放进仓库根目录。
    • 主页文件必须为 index.html
    • index.html 必须在仓库根目录
  3. 提交并推送到 GitHub： 
    git add .
git commit -m "初始化网站"
git push

四、访问你的网站

在浏览器输入：
https://你的用户名.github.io
一般 1-10 分钟内网站即可上线。如未上线请稍等片刻并刷新页面。

五、自定义域名（可选）

  1. 在仓库根目录新建名为 CNAME 的文本文件，文件内容为你的自定义域名（如 www.example.com）。
  2. 在域名服务商处设置 CNAME 解析，将 www.example.com 指向 你的用户名.github.io。
    • 等待 DNS 生效（通常几分钟到 24 小时）
  3. 推送 CNAME 文件，GitHub Pages 会自动启用你的域名。

六、常见问题与排查

  • Q: 网站打不开？
    A: 检查仓库名、index.html 位置，确认已推送到 GitHub。
  • Q: 页面长时间未更新？
    A: 清理浏览器缓存，或在仓库 Settings → Pages 页面点击 Rebuild。
  • Q: 自定义域名不生效？
    A: 检查 CNAME 文件内容和域名解析，等待 DNS 生效。
  • Q: 如何开启 HTTPS？
    A: GitHub Pages 会自动为所有自定义域名提供免费 HTTPS。
